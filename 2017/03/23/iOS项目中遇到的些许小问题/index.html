很多属性文字显示需要准确高度的问题

  写代码时，发现属性文字的高度计算很难过，因为属性很多 ，一段段的计算也很麻烦，查询之后只有sizetofit和sizeThatFits，发现两个在cell的使用中都有问题，就是滑动完成时label会不断调整大小，而且很奇怪，于是用sizeThatFits（0，0），发现就按一行显示，翌日处理bug时，想到x,y是不是优先调整不接近预设的标准，于是将x设为预期宽度，y设为0，然后就得到预期效果。

应用缓存的问题

tmp:提供一个即时创建临时文件的地方：关闭app后自动清除，app更新后会被清除;
Library：存储程序的默认设置或其它状态信息：在内存不足的情况下有可能会清除；
里面又包含两个文件夹Caches和Preference；
Caches，存放缓存文件，iTunes不会备份此目录,
Documents：苹果建议将程序中建立的或在程序中浏览到的文件数据保存在该目录下，iTunes备份和恢复的时候会包括此目录。

新浪微博个人主页共同简单header的思路

  其实很简单，在添加tableview之后，直接在view上添加一个view当做tableview的view。然后对于view采用 hitTest:poinwithEvent:的方法，将响应事件事件向下传递，即可。
例：- (UIView )hitTest:(CGPoint)point withEvent:(UIEvent )event {
UIView *result = [super hitTest:point withEvent:event];
if (result == self) {
return nil;
} else {
return result;
}
}
（注：此方法有问题，即view上的subview中如果有手势操作，如果不处理则无法返回给view。）

有特殊字符的空格处理

原因：空格含有两个ASIIC码，一种值是32，一种值是160的。
解决方法：1、直接把空格粘贴过来；2、[[string componentsSeparatedByCharactersInSet:[[NSCharacterSet characterSetWithCharactersInString:@”0123456789”] invertedSet]] componentsJoinedByString:@””]。

应用间的跳转问题

  苹果对于应用间的跳转可采用openurl的形式，对于后面参数的拼接可以达到各种跳转（前提是跳转的应用能解析出参数并且实现后续跳转的方法），而对于应用白名单和url types的理解：首先在url types里设置App的appid，App打开时，根据第三方服务器的规则将App Key 和 App Secret拼接以便需要时验证身份。通过设置白名单可在跳转第三方服务器保证可以执行服务。
